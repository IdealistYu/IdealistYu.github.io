「グラフの理論」のまとめ

Shortest Path（最短経路）

Finding Shortest-Path in Unweighted Graphs（幅優先探索）

广度优先搜索（BFS）是一种用于无权图的最短路径算法。它是一种基于队列的算法，用于从图中的一个起始顶点开始遍历，找到到达所有其他顶点的最短路径。

BFS算法的基本思想是从起始顶点开始，先将其加入队列中，然后从队列中取出一个顶点并遍历它的所有邻居顶点，将未被访问的邻居顶点加入队列中。然后重复这个过程，直到队列为空为止。在遍历过程中，需要记录每个顶点的距离和它的前驱节点，以便计算出到达每个顶点的最短路径。

BFS算法的流程如下：

  1. 从起始顶点开始，将其加入队列中，并标记为已访问。
  2. 从队列中取出一个顶点，并遍历它的所有邻居顶点。对于每个未被访问过的邻居顶点，将其加入队列中，并标记为已访问。同时记录它的距离和前驱节点。
  3. 重复步骤2，直到队列为空。

BFS算法的时间复杂度为 \(O(V+E)\)，其中 \(V\) 是顶点数，\(E\) 是边数。因为算法需要遍历每个顶点和每条边。

需要注意的是，BFS算法仅适用于无权图。如果图中存在边的权重，那么应该使用其他算法，如Dijkstra算法或A*算法，来计算最短路径。

Finding Shortest-Path in Weighted Graphs（ダイクストラのアルゴリズム）

Dijkstra算法是一种用于有权图的最短路径算法。它是一种贪心算法，用于从图中的一个起始顶点开始遍历，找到到达所有其他顶点的最短路径。

Dijkstra算法的基本思想是维护一个距离表，用于记录从起始顶点到每个顶点的距离。起始时，距离表中除了起始顶点，其他所有顶点的距离均设为无穷大。然后从距离表中选择距离最小的顶点，并将其标记为已访问。然后更新距离表，将所有从该顶点出发的边所连接的顶点的距离更新为从起始顶点到该顶点的距离与该边的权重之和的最小值。然后重复这个过程，直到所有顶点都被标记为已访问。

Dijkstra算法的流程如下：

  1. 初始化距离表，将起始顶点的距离设为0，其他顶点的距离设为无穷大。
  2. 从距离表中选择距离最小的顶点，并将其标记为已访问。
  3. 遍历所有从该顶点出发的边，对于每条边，将连接的顶点的距离更新为从起始顶点到该顶点的距离与该边的权重之和的最小值。
  4. 重复步骤2和3，直到所有顶点都被标记为已访问。

Dijkstra算法的时间复杂度为 \(O(V^2)\)，其中 \(V\) 是顶点数。如果使用堆来实现距离表，可以将时间复杂度优化到 \(O(E\log V)\)，其中 \(E\) 是边数。

需要注意的是，Dijkstra算法仅适用于没有负权边的图。如果图中存在负权边，那么应该使用其他算法，如Bellman-Ford算法或A*算法，来计算最短路径。

Minimum Spanning Trees（最小全域木）

Prim's Algorithm（プリムのアルゴリズム）

Prim算法是一种用于无向加权图的最小生成树算法，它的基本思想是从一个顶点开始，逐步扩展生成树，直到所有顶点都被包含在生成树中为止。

Prim算法的流程如下：

  1. 选取任意一个顶点作为起始顶点，并将它加入生成树中。
  2. 从生成树中的所有顶点出发，遍历它们的所有邻居顶点，找到连接生成树和非生成树顶点之间权重最小的边。
  3. 将权重最小的边所连接的顶点加入生成树中，并将该边加入最小生成树的边集合中。
  4. 重复步骤2和3，直到生成树中包含所有顶点。

Prim算法的时间复杂度为 \(O(E\log V)\)，其中 \(V\) 是顶点数，\(E\) 是边数。因为算法需要遍历所有边并在每次遍历时对堆进行一次操作，堆的操作次数为 \(E\log V\)。

需要注意的是，Prim算法仅适用于无向加权图。如果图中存在负权边，那么应该使用其他算法，如Kruskal算法，来计算最小生成树。

Kruskal's Algorithm（クラスカルのアルゴリズム）

Kruskal算法是一种用于无向加权图的最小生成树算法，它的基本思想是从所有边中按照权重从小到大的顺序选取边，依次将其加入生成树中，如果加入该边不会形成环，则将其加入最小生成树的边集合中。

Kruskal算法的流程如下：

  1. 将图中所有边按照权重从小到大的顺序排序。
  2. 从权重最小的边开始遍历，如果加入该边不会形成环，则将该边加入最小生成树的边集合中。
  3. 判断加入该边后是否会形成环，可以使用并查集来实现。如果加入该边后形成了环，则不加入该边。
  4. 重复步骤2和3，直到最小生成树包含了所有顶点。

Kruskal算法的时间复杂度为 \(O(E\log E)\)，其中 \(E\) 是边数。因为算法需要对所有边进行排序，并在每次加入边时进行一次并查集的操作，因此时间复杂度为 \(O(E\log E)\)。

需要注意的是，Kruskal算法仅适用于无向加权图。如果图中存在负权边，那么应该使用其他算法，如Prim算法，来计算最小生成树。

Maximum Flow Problem（最大流問題）

Ford-Fulkerson Algorithm（フォード-ファルカーソン法）

Ford-Fulkerson算法是求解最大流问题的一种基本方法。该算法通过不断寻找增广路径，并沿着增广路径更新网络流，直到不存在增广路径为止。下面是一般情况下的算法流程：

  1. 初始化网络流为0。
  2. 在残留网络中寻找一条增广路径（可以使用深度优先搜索或广度优先搜索），如果能找到增广路径，则计算增广路径上的最小容量（也称为瓶颈容量）。
  3. 沿着增广路径更新网络流，将增广路径上的每一条边的流量增加最小容量，同时将反向边的流量减少相同的值。
  4. 重复步骤2和步骤3，直到没有增广路径为止。此时，网络流的值就是最大流的值。

在该算法中，残留网络是一个与原网络具有相同的拓扑结构，但是每条边的容量是当前流和最大容量的差值。如果某个边的容量为0，说明该边不能再传输流量，因此在增广路径的选择过程中，需要排除容量为0的边。

Ford-Fulkerson算法的时间复杂度取决于增广路径的选择方式。如果使用深度优先搜索，最坏情况下时间复杂度可以达到 \(O(Ef)\)，其中 \(E\) 是边的数量，\(f\) 是最大流的值。需要注意的是，Ford-Fulkerson算法本身不是一个具体的算法，而是一种解决最大流问题的基本思想。实际上，Ford-Fulkerson算法的具体实现方法有很多种，比较常见的有Edmonds-Karp算法和Dinic算法。

Edmonds-Karp算法和Dinic算法都是Ford-Fulkerson算法的优化版本。

Edmonds-Karp算法通过使用广度优先搜索来找到增广路径，而不是深度优先搜索。这样做可以保证在找到增广路径时，路径的长度最短，因此算法的时间复杂度是O(VE^2)，其中V是节点数，E是边数。

Dinic算法则使用一种叫做层次图的数据结构来寻找增广路径，它通过对图进行分层，使得在每次寻找增广路径时只需要在同一层次内进行搜索。这样做可以显著减少搜索的次数，从而使算法的时间复杂度为O(V^2E)，其中V是节点数，E是边数。

相比之下，Ford-Fulkerson算法并没有明确的增广路径寻找策略，而是依靠深度优先搜索来寻找增广路径。这种方法的时间复杂度并不好估计，因为它取决于寻找增广路径的方式和具体的输入数据。

因此，尽管三种算法都是求解最大流问题的经典算法，但它们的具体实现和时间复杂度上存在差异。选择哪一种算法应该根据具体情况而定。

Maximum-Cardinality Bipartite Matching（二部グラフの最大マッチング）

Hungrian Algorithm（ ハンガリー法）

匈牙利算法是一种解决二分图最大匹配问题的经典算法。二分图是指一个图的所有顶点可以分为两部分，使得每条边的两个端点分别属于不同的部分。而匈牙利算法的目标就是在一个二分图中找到一个最大匹配，即最大的边集合，其中每个顶点最多与一条边相连。

下面是匈牙利算法的步骤：

  1. 对于二分图中的每个未匹配点，尝试从该点出发找到增广路，即一条交替经过未匹配边和已匹配边的路径，路径的起点和终点都是未匹配点。

  2. 如果存在增广路，则将该路径上的已匹配边和未匹配边交换，即可得到一个新的匹配。

  3. 如果不存在增广路，则当前匹配就是最大匹配，算法结束。

  4. 重复步骤1-3，直到找到所有增广路或者确定了最大匹配。

匈牙利算法的时间复杂度为 O(mn)，其中 m 和 n 分别为二分图的左右两部分的顶点数，因此该算法适用于小规模的二分图。如果需要处理大规模的图，则需要使用更高效的算法。

Reduce MCBM to the max-flow problem（最大フロー問題を使った解法）

最大流问题算法可以用来解决二分图最大匹配问题，其中最经典的算法是Ford-Fulkerson算法和Edmonds-Karp算法。这里介绍一下使用Ford-Fulkerson算法解决二分图最大匹配问题的流程：

  1. 将二分图转化为网络流图：将二分图的左侧顶点设为源点S，右侧顶点设为汇点T，每条边的容量设为1，得到一个网络流图。

  2. 在网络流图上运行Ford-Fulkerson算法，求出最大流量。

  3. 根据最大流的值，构造一个最大匹配：对于每条从左侧顶点到右侧顶点的边，如果这条边的流量为1，那么就说明这是一个匹配边。将所有的匹配边集合起来，就构成了一个最大匹配。

Ford-Fulkerson算法的时间复杂度为O(f * E)，其中f表示最大流量，E表示网络流图的边数。在二分图最大匹配问题中，f的值最大为n，即左侧顶点数，因此算法的时间复杂度为O(nE)。

需要注意的是，最大流算法可以解决一般的网络流问题，因此当二分图不是完全二分图（即有左侧顶点和右侧顶点之间的边）时，最大流算法仍然可以求解，但得到的最大流不一定是最大匹配。此时需要对得到的最大流进行调整，才能得到一个最大匹配。

Stable Marriage Problem（安定結婚問題）

Gale-Shapley Algorithm（ゲール-シャプレイ アルゴリズム）

稳定婚姻问题的背景是这样的：有n个男人和n个女人，每个人都有一个偏好列表，其中男人对女人的偏好按照从高到低的顺序排序，女人对男人的偏好也是如此。现在需要进行匹配，每个男人只能与一个女人结婚，每个女人也只能与一个男人结婚。如果存在一对男女，他们之间有更高的偏好关系，但是目前却没有匹配在一起，那么就称这个匹配是不稳定的。

Gale-Shapley算法的核心思想是：让每个男人按照自己的偏好列表依次向自己最喜欢的女人求婚，同时让每个女人选择自己最喜欢的求婚者。如果一个女人已经有了婚约，她会比较自己的现任丈夫和新的求婚者，如果新的求婚者更好，她会拒绝现任丈夫，选择新的求婚者。如果一个男人被拒绝了，他就会去向自己的次喜欢女人求婚，直到他找到了一个女人愿意和他结婚。使用Gale-Shapley Algorithm计算稳定婚配问题的步骤如下：

  1. 每个男性向他最喜欢的女性求婚，每个女性先暂时保留她最喜欢的男性的求婚。

  2. 对于每个女性，如果她已经被求婚，那么她会比较自己当前的男性和新的求婚者，选择她更喜欢的男性，并将之前的男性归还到未婚状态。

  3. 重复步骤2，直到所有女性都有一个男性。

  4. 对于每个男性，如果他被拒绝了，那么他向下一个最喜欢的女性求婚，并重复步骤2和3，直到他找到一个女性接受他的求婚。

  5. 当所有男性都有一个女性时，算法结束。

通过这个过程，每个男性和女性都会得到一个稳定的婚配，也就是说，不存在两个人，其中一个更喜欢另一个，但是被配对到了不喜欢的人身上。Gale-Shapley Algorithm的时间复杂度为O(n^2)，其中n是男女数量的总和。

Euler circuit & Hamiltonian cycle（オイラー回路とハミルトン閉路）

欧拉回路是指一条经过所有边恰好一次的回路。注意，欧拉回路不需要经过所有的节点，只需要经过所有的边即可。如果图中存在欧拉回路，那么这个图被称为欧拉图。

相比之下，哈密顿闭路则是指一条经过所有节点恰好一次的回路。也就是说，哈密顿闭路必须经过所有节点，但不需要经过所有边。如果图中存在哈密顿闭路，那么这个图被称为哈密顿图。

在实际应用中，欧拉回路和哈密顿闭路都具有一定的意义。比如说，在城市旅游规划中，欧拉回路可以帮助规划出一条游览所有景点的最短路径；而哈密顿闭路可以帮助规划出一条游览所有景点的路径，且不需要重复经过某个景点。

需要注意的是，判断一个图是否存在欧拉回路或哈密顿闭路是一个NP完全问题，即当前不存在有效的多项式时间算法。因此，只能使用一些启发式算法或近似算法进行求解。

Euler circuit（オイラー回路）

欧拉回路相关定理：

  1. 偶度节点定理：一个无向图存在欧拉回路，当且仅当该图的所有节点度数均为偶数。

  2. 一个无向图存在欧拉通路，当且仅当该图的所有节点度数均为偶数或者有且仅有两个节点的度数为奇数。

  3. 东京-底特律定理：一个无向图存在欧拉回路，当且仅当该图的任意两个顶点之间都存在偶数条路径。

  4. 奥雷定理：一个有向图存在欧拉回路，当且仅当该图的所有节点的入度和出度相等。

Hamiltonian cycle（ハミルトン閉路）

哈密顿闭路相关定理：

  1. Dirac（ディラック）定理：设\(G\)是一个有\(n\)个节点的简单图，如果对于任意一个节点\(v\in G\)，其度数\(d(v)\geq \frac{n}{2}\)，那么该图存在哈密顿闭路。
  2. Ore（オア）定理：设\(G\)是一个有\(n\)个节点的简单图，如果对于任意两个不相邻的节点\(u,v\in G\)，都满足\(d(u)+d(v)\geq n\)，那么该图存在哈密顿闭路。

Note（注意）

需要注意的是，这些定理只是判断欧拉回路或哈密顿闭路是否存在的充分条件，并不是必要条件。在实际求解问题时，我们需要综合考虑这些定理以及具体的算法实现。

Graph Coloring（グラフの彩色）

图的着色是指在给定的图中为每个顶点分配颜色，使得相邻的顶点颜色不同的问题。可以把图的着色问题看成一种优化问题，即在最小的颜色数的前提下，使得图的每个顶点都拥有一种不同的颜色。

图的着色是图论中的一个重要问题，被广泛应用于调度、分配、排程等问题中。与此相关的还有很多定理和算法，例如著名的四色定理，以及Welsh-Powell算法和DSATUR算法等启发式算法。由于图的着色问题通常是一个NP难问题，因此在实际应用中通常需要使用近似算法。

vertex coloring（頂点彩色）

  1. ブルックス（Brooks）の定理：連結グラフ\(G\)が完全グラフでもなく奇数長閉路でもないならば,\(G\)は\(\Delta(G)\)-頂点彩色可能である。
  2. 正則でない連結グラフ\(G\)は,\(\Delta(G)\)-頂点彩色可能である。
  3. \(G\)が\(2\)-頂点彩色可能であるための必要十分条件は,\(G\)が\(2\)部ブラフであることである。
  4. \(G\)が\(2\)-頂点彩色可能であるための必要十分条件は,\(G\)が奇数長閉路を持たないことである。
  5. 任意の平面グラフは\(5\)-頂点彩色可能である。

edge coloring（辺彩色）

  1. ビジング（Vizing）の定理：任意のグラフ\(G\)に対して\(\chi^{\prime}(G)\leq\Delta(G)+1\)が成り立つ。
  2. ケーニッヒ（König）の定理：任意の\(2\)部グラフ\(G\)に対して\(\chi^{\prime}(G)\leq\Delta(G)\)が成り立つ。
